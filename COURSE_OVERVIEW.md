# Course Overview

## Course framing

Digital disruption is not a buzzword. It is a structural shift in how value is created, delivered, captured, and defended. This course studies those shifts through a strategic and consulting lens.

Students will learn to:

- distinguish disruption from ordinary technological change,
- analyze industries under digital pressure,
- diagnose where incumbents are vulnerable,
- compare strategic responses,
- design transformation recommendations grounded in evidence.

## Core intellectual anchors

This course uses three primary conceptual anchors:

### 1. Rogers' Digital Transformation Roadmap
Rogers provides the five-domain model used throughout the course:
- customers
- competition
- data
- innovation
- value

### 2. Christensen's disruption framework
Christensen, Raynor, and McDonald help us distinguish:
- low-end disruption
- new-market disruption
- sustaining innovation

### 3. Yoo, Lyytinen, and Boland's digital innovation architecture
This academic lens explains why digital innovation behaves differently from earlier industrial innovation through:
- layered architecture
- reprogrammability
- generativity
- distributed innovation

## Supporting tools

The course also uses practical analysis methods:
- issue trees
- hypothesis logs
- Porter's Five Forces
- value chain analysis
- jobs-to-be-done framing
- strategic problem statements

## Semester project

Each team works on one client industry context across the semester. Weekend 1 launches the project. Later weekends deepen analysis and build toward transformation strategy and implementation logic.

## Weekend 1 outputs

By the end of Weekend 1, teams should have:
- a team working agreement,
- a strategic problem statement,
- an issue tree,
- a hypothesis log v1,
- an industry map,
- a competitive landscape v1,
- a sprint plan for March 15 to April 4.

These are not filler artifacts. They become the raw material for later deliverables.
